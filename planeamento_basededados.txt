PARTE 2 — DESENHO DO ESQUEMA DE BASE DE DADOS
========================================================================
Tabelas a criar (Postgres, schema "public"):

2.1. categories
     - id            uuid PK default gen_random_uuid()
     - slug          text UNIQUE NOT NULL      (ex: 'corrida')
     - label         text NOT NULL             (ex: 'Corrida')
     - created_at    timestamptz default now()

2.2. products
     - id              uuid PK default gen_random_uuid()
     - name            text NOT NULL
     - subtitle        text
     - category_id     uuid NOT NULL REFERENCES categories(id)
     - price           numeric(10,2) NOT NULL CHECK (price > 0)
     - original_price  numeric(10,2) CHECK (original_price IS NULL OR original_price > price)
     - bg              text                     (gradiente CSS de fallback)
     - image_path      text                     (caminho no Supabase Storage)
     - description     text
     - sizes           text[] NOT NULL DEFAULT '{}'
     - is_new          boolean NOT NULL DEFAULT false
     - featured        boolean NOT NULL DEFAULT false
     - stock           integer NOT NULL DEFAULT 0 CHECK (stock >= 0)
     - active          boolean NOT NULL DEFAULT true   (soft-delete / esconder da loja)
     - created_at      timestamptz DEFAULT now()
     - updated_at      timestamptz DEFAULT now()

2.3. customers  (criado automaticamente no 1º checkout; sem login de cliente)
     - id            uuid PK default gen_random_uuid()
     - name          text NOT NULL
     - email         text NOT NULL
     - address       text NOT NULL
     - zip           text NOT NULL
     - city          text NOT NULL
     - created_at    timestamptz DEFAULT now()

2.4. orders
     - id              uuid PK default gen_random_uuid()
     - reference       text UNIQUE NOT NULL     (ex: 'AC482913', gerado por trigger)
     - customer_id     uuid NOT NULL REFERENCES customers(id)
     - subtotal        numeric(10,2) NOT NULL
     - shipping        numeric(10,2) NOT NULL DEFAULT 0
     - total           numeric(10,2) NOT NULL
     - status          text NOT NULL DEFAULT 'pago'
                        CHECK (status IN ('pendente','pago','enviado','entregue','cancelado'))
     - created_at      timestamptz DEFAULT now()

2.5. order_items
     - id              uuid PK default gen_random_uuid()
     - order_id        uuid NOT NULL REFERENCES orders(id) ON DELETE CASCADE
     - product_id      uuid REFERENCES products(id) ON DELETE SET NULL
     - product_name    text NOT NULL      (snapshot — mantém-se mesmo se o produto for apagado)
     - size             text
     - unit_price       numeric(10,2) NOT NULL
     - qty              integer NOT NULL CHECK (qty > 0)

2.6. newsletter_subscribers  (para tornar o formulário de newsletter real)
     - id            uuid PK default gen_random_uuid()
     - email         text UNIQUE NOT NULL
     - created_at    timestamptz DEFAULT now()

2.7. job_applications  (para tornar carreiras.html funcional, opcional)
     - id            uuid PK default gen_random_uuid()
     - job_title     text NOT NULL
     - name          text NOT NULL
     - email         text NOT NULL
     - phone         text
     - message       text
     - cv_path       text                (caminho no Storage, se anexar CV)
     - created_at    timestamptz DEFAULT now()

Nota: autenticação do admin usa o sistema nativo "auth.users" do Supabase
(ver PARTE 5) — não é preciso criar tabela de utilizadores manual.

PARTE 3 — SQL DE CRIAÇÃO (executar no SQL Editor do Supabase, por ordem)
========================================================================
Ir a "SQL Editor" no painel Supabase → "New query" → colar e correr cada
bloco abaixo, um de cada vez, na ordem apresentada.

--- 3.1 Extensões necessárias ---
create extension if not exists "pgcrypto";

--- 3.2 Categorias ---
create table categories (
  id uuid primary key default gen_random_uuid(),
  slug text unique not null,
  label text not null,
  created_at timestamptz default now()
);

insert into categories (slug, label) values
  ('corrida', 'Corrida'),
  ('treino', 'Treino'),
  ('futebol', 'Futebol'),
  ('estilo', 'Estilo');

--- 3.3 Produtos ---
create table products (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  subtitle text,
  category_id uuid not null references categories(id),
  price numeric(10,2) not null check (price > 0),
  original_price numeric(10,2) check (original_price is null or original_price > price),
  bg text,
  image_path text,
  description text,
  sizes text[] not null default '{}',
  is_new boolean not null default false,
  featured boolean not null default false,
  stock integer not null default 0 check (stock >= 0),
  active boolean not null default true,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create index idx_products_category on products(category_id);
create index idx_products_active on products(active);

-- trigger para manter updated_at atualizado
create or replace function set_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger trg_products_updated_at
before update on products
for each row execute function set_updated_at();

--- 3.4 Clientes ---
create table customers (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  email text not null,
  address text not null,
  zip text not null,
  city text not null,
  created_at timestamptz default now()
);

--- 3.5 Encomendas ---
create table orders (
  id uuid primary key default gen_random_uuid(),
  reference text unique not null,
  customer_id uuid not null references customers(id),
  subtotal numeric(10,2) not null,
  shipping numeric(10,2) not null default 0,
  total numeric(10,2) not null,
  status text not null default 'pago'
    check (status in ('pendente','pago','enviado','entregue','cancelado')),
  created_at timestamptz default now()
);

-- gera referência tipo AC482913 automaticamente
create sequence order_ref_seq;

create or replace function set_order_reference()
returns trigger as $$
begin
  new.reference := 'AC' || lpad(nextval('order_ref_seq')::text, 6, '0');
  return new;
end;
$$ language plpgsql;

create trigger trg_orders_reference
before insert on orders
for each row execute function set_order_reference();

--- 3.6 Itens da encomenda ---
create table order_items (
  id uuid primary key default gen_random_uuid(),
  order_id uuid not null references orders(id) on delete cascade,
  product_id uuid references products(id) on delete set null,
  product_name text not null,
  size text,
  unit_price numeric(10,2) not null,
  qty integer not null check (qty > 0)
);

create index idx_order_items_order on order_items(order_id);

--- 3.7 Newsletter ---
create table newsletter_subscribers (
  id uuid primary key default gen_random_uuid(),
  email text unique not null,
  created_at timestamptz default now()
);

--- 3.8 Candidaturas (carreiras) ---
create table job_applications (
  id uuid primary key default gen_random_uuid(),
  job_title text not null,
  name text not null,
  email text not null,
  phone text,
  message text,
  cv_path text,
  created_at timestamptz default now()
);

PARTE 4 — SEGURANÇA: ROW LEVEL SECURITY (RLS)
========================================================================
Como a chave "anon" fica exposta no código do site, é OBRIGATÓRIO ativar
RLS em todas as tabelas e definir políticas explícitas. Sem isto,
qualquer visitante poderia ler/escrever tudo diretamente pela API.

Correr no SQL Editor:

--- 4.1 Ativar RLS em todas as tabelas ---
alter table categories enable row level security;
alter table products enable row level security;
alter table customers enable row level security;
alter table orders enable row level security;
alter table order_items enable row level security;
alter table newsletter_subscribers enable row level security;
alter table job_applications enable row level security;

--- 4.2 Categorias: leitura pública, escrita só para admin autenticado ---
create policy "categorias visiveis a todos"
  on categories for select
  using (true);

create policy "categorias so admin escreve"
  on categories for all
  using (auth.role() = 'authenticated')
  with check (auth.role() = 'authenticated');

--- 4.3 Produtos: leitura pública (só ativos), escrita só admin ---
create policy "produtos ativos visiveis a todos"
  on products for select
  using (active = true or auth.role() = 'authenticated');

create policy "produtos so admin insere"
  on products for insert
  with check (auth.role() = 'authenticated');

create policy "produtos so admin atualiza"
  on products for update
  using (auth.role() = 'authenticated')
  with check (auth.role() = 'authenticated');

create policy "produtos so admin apaga"
  on products for delete
  using (auth.role() = 'authenticated');

--- 4.4 Clientes: qualquer pessoa pode criar o seu registo no checkout,
--- mas ninguém (exceto admin) pode ler dados de outros clientes ---
create policy "qualquer um cria cliente no checkout"
  on customers for insert
  with check (true);

create policy "so admin le clientes"
  on customers for select
  using (auth.role() = 'authenticated');

--- 4.5 Encomendas: criação pública (checkout), leitura só admin ---
create policy "qualquer um cria encomenda"
  on orders for insert
  with check (true);

create policy "so admin le encomendas"
  on orders for select
  using (auth.role() = 'authenticated');

create policy "so admin atualiza encomendas"
  on orders for update
  using (auth.role() = 'authenticated')
  with check (auth.role() = 'authenticated');

--- 4.6 Itens de encomenda: mesma lógica ---
create policy "qualquer um cria itens de encomenda"
  on order_items for insert
  with check (true);

create policy "so admin le itens de encomenda"
  on order_items for select
  using (auth.role() = 'authenticated');

--- 4.7 Newsletter: qualquer um subscreve, só admin lê a lista ---
create policy "qualquer um subscreve newsletter"
  on newsletter_subscribers for insert
  with check (true);

create policy "so admin le newsletter"
  on newsletter_subscribers for select
  using (auth.role() = 'authenticated');

--- 4.8 Candidaturas: qualquer um candidata-se, só admin lê ---
create policy "qualquer um se candidata"
  on job_applications for insert
  with check (true);

create policy "so admin le candidaturas"
  on job_applications for select
  using (auth.role() = 'authenticated');

     const ADMIN_USERNAME = 'adriano';
     const ADMIN_EMAIL = 'adriano@adrianoscloset.pt';

     async function loginAdmin(username, password) {
       if (username.trim().toLowerCase() !== ADMIN_USERNAME) {
         throw new Error('Utilizador ou password inválidos.');
       }
       const { data, error } = await supabaseClient.auth.signInWithPassword({
         email: ADMIN_EMAIL,
         password
       });
       if (error) throw new Error('Utilizador ou password inválidos.');
       return data.session;
     }

     async function logoutAdmin() {
       await supabaseClient.auth.signOut();
     }

     async function requireAdminSession() {
       const { data: { session } } = await supabaseClient.auth.getSession();
       if (!session) {
         window.location.href = 'login.html';
       }
       return session;
     }

PARTE 7 — REESCREVER js/db.js PARA USAR SUPABASE (em vez de localStorage)
========================================================================
Substituir o objeto DB atual por chamadas assíncronas ao Supabase,
mantendo os mesmos nomes de método para minimizar alterações em
store.js/admin.js (que passam a usar "await"):

  DB.init()                 → deixa de ser necessário (não há seed local)
  DB.getProducts()          → supabaseClient.from('products')
                                 .select('*, categories(slug,label)')
                                 .eq('active', true)
                                 .order('created_at')
  DB.getProduct(id)         → .select(...).eq('id', id).single()
  DB.saveProduct(product)   → upsert (insert se sem id, update se tiver id)
  DB.deleteProduct(id)      → update active=false (soft delete) EM VEZ DE
                                 apagar a linha — mantém histórico em
                                 order_items que referenciam o produto
  DB.saveOrder(order)       → 1) insert em customers → obter customer_id
                               2) insert em orders (subtotal, shipping, total, customer_id)
                               3) insert em order_items (um por item do carrinho)
                               tudo dentro da mesma função, sequencialmente
  DB.getOrders()            → supabaseClient.from('orders')
                                 .select('*, customers(*), order_items(*)')
                                 .order('created_at', { ascending: false })

Todas as funções que chamam DB.* em store.js e admin.js passam a ser
"async function" e a usar "await DB.xxx(...)".
